Competition manual

Initial steps

  1. Scans -- system and network
    2. nmap
  2. Access controls / user privileges
    3. Update roles, rotate passwords and keys
  3. Rules for White Team
    4. Create special, separate rules to protect white team access
  4. Local firewalls
    5. iptables for Linux
    Windows Defender Firewall for Windows
  5. Windows: make sure Defender is running and is up to date
  6. Linux: make sure PAM is installed and running
  7. SSL and SSH
    8. Verify certificates
  8. Get running services & disable unused services
    9. Windows
    Linux

Master Protocol

  1. Loop 2-6.
  2. Write down each task completed on your machine/s as you go for reference.
  3. Work on specific machine tasks (Windows, Linux, Firewall, etc.)
  4. Read and work on hardening tasks for your machine.
  5. Call out that you are "finished with maintenance".
  6. Ask if anyone needs help with maintenance.
  7. Look for new injects or continue working on previous injects.
  8. Ask if anyone needs help with injects.

Incident Response

  1. Workstations will call out your name when your service goes down.
  2. Work with captain to determine if incident is internal (you) or external (red team) caused.
  3. Check running services and processes -- kill anything that shouldn't be running, try and start anything that should be running.
  4. For unwanted service/process status -- remove user or update passwords.
  5. Work with captain (and Google) to determine how to return services to proper state.
  6. Write down incident and responses to return to service; if you cannot return to service, call out for help, share record.
